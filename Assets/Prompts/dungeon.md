유니티로 제작하는, 그리드 위에서 작동하는 텍스트 위주 던전 크롤링 게임의 코드를 작성한다. 
익스트랙션 게임의 문법을 사용하여, 던전에 아이템을 챙겨 들어가 탐사 후 전리품을 획득하여 나오고, 만약 실패한다면 들고 간 아이템과 획득한 아이템을 모두 잃게 된다.

여기서는 그 게임의 한번의 세션에 해당하는 "던전"을 작성한다.

GDD는 `dungeon-gdd.md` 파일을 참조한다.
단, 확정된 GDD가 아니라 아이디어의 모음 정도이며, 프로토타이핑 중이기 때문에 언제든 바뀔 수 있다. 따라서 단순 참고용으로만 사용한다.
GDD와 입력이 상반되거나 모순될 경우 입력을 우선한다.


던전에 대한 정적 데이터는 다음과 같은 정보가 들어 있어야 한다.
* 크기 (가로, 세로)
* 그리드 전체에 대한 타일 데이터

던전에 대한 동적 데이터는 다음과 같은 정보가 있다.
* 각 셀에 대한 시야 상태 (모름, 보임, 알지만 지금은 보이지 않음)-일반적인 fog of war이다.

각 셀에 대한 데이터는 다음과 같다
* 타입
  * 타입에는 빈공간, 적, 아이템(전리품), 정보 공간(시야를 밝히는 등의 특정 보너스), 휴식 공간(체력 회복 등), 탈출 지점 등이 있다.
  * 각 타일 타입에 대한 정적 및 동적 데이터는 다음과 같다.
  * 적
    * 정적 : 적의 ID
    * 동적 : 적의 상태 (살았다/죽었다 등)
  * 아이템
    * 정적 : 아이템 ID
    * 동적 : 상태 (닫힘/열림 등)
  * 정보 공간
    * 정적 : 타입 ID (현재 세부 기획 없음)
    * 동적 : 사용했는지 여부
  * 휴식 공간
    * 정적 : 타입 ID (현재 세부 기획 없음)
    * 동적 : 사용했는지 여부
